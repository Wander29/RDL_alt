Algoritmo md5 in PHP

In questa esperienza di laboratorio l’obiettivo è quello di creare un pagina php che generi un digest di una stringa inserita dall’utente tramite un normale form html.

Configurazione

Per prima cosa si deve avviare easyPHP e inizializzare il server http, il web server locale che verrà utilizzato dalla pagina php.

Una volta operativo si passa ad avviare l’IDE da utilizzare, NetBeans.
Da NetBeans si crea un nuovo progetto (File -> New Project) e si sceglie PHP – PHP application.

Successivamente si deve dare il nome al progetto. Nella stessa schermata è di fondamentale importanza fare attenzione alla directory nel quale esso verrà salvato.

Il modo migliore e più efficiente di salvare progetti PHP è quello di salvarli all’interno del web server in modo da poter elaborare e visualizzare il risultato immediatamente.

Per fare ciò si deve conoscere il percorso di installazione del web server e la cartella da utilizzare. Questa si chiama scripts è solitamente si trova al seguente percorso: C:\Program Files (x86)\EasyPHP-DevServer-14.1VC11\data\localweb\scripts

Cliccando su Browse si può selezionare tale cartella:

Una volta selezionata si crea la cartella del progetto dentro essa e si può proseguire. Nella prossima schermata si deve fare attenzione al path nell’indirizzo che compare (relativo al web server).

Solitamente l’indirizzo è: http://localhost/{cartella_progetto}/

È importante notare che tale indirizzo è sbagliato poiché è necessario aggiungere, manualmente, la cartella scripts
Pertanto il giusto path è:
http://localhost/scripts/{cartella_progetto}/
Si deve ricordare che chiamando un file 'index.php' esso verrà aperto all'apertura della cartella dal web server, senza bisogno di specificare il suo nome.

A questo punto la configurazione è terminata.

E' presente l'attributo action che indica verso quale pagina php si devono inviare i dati del form, in questo caso è vuoto poichè indica la stessa pagina in questione, e l'attributo method che specifica se utilizzare il POST od il GET per l'invio dei dati al server PHP.

L'attributo name dell'input tipo testo sarà l'identificativo con cui nello script php identificheremo lo stesso per ottenere i dati contenuti in esso:

E' infine presente un bottone submit nel tag input che quando verrà cliccato invierà i dati al server.
Il bottone reset è una piccola aggiunta all'esercizio originale che tramite una semplicissima funzione javascript svuota il value dell'input chiamato testo.

Lo script PHP inizia con un piccolo controllo sull' input: nel caso non sia stata inserito nulla da tastiera lo script non svolge nessuna funzione, in caso contrario eseguirà la cifratura, obiettivo di questa esercitazione.

Per prima cosa viene assegnato ad una variabile il contenuto dell'input testo ottenuto tramite POST una volta clicato il pulsante Cifra (submit).
Successivamente viene assegnata ad una seconda variabile il valore della prima, cifrata in MD5, tramite la funzione md5(); essa conterrà quindi il digest dell'input inserito da tastiera.

Infine vengono stampate nell'HTML entrambe le variabili tramite echo (i punti indicano la concatenazione tra stringhe, in PHP).

Conclusioni
